## 🎯 关键改进

### 1. 📱 前端改进

#### 新增功能
- **任务调度系统**：基于 `AutoRefreshCache` 和 `ScreenMonitorTask` 的高级任务调度
- **进程监控**：使用 `pidusage` 监控应用资源使用情况
- **事件总线**：使用 `mitt` 简化跨组件通信

#### 构建优化
- 改进的跨平台 Python 构建脚本
- 支持 `copy-backend` 预构建后端
- Windows 平台自动跳过 macOS 特定工具构建

### 2. 🐍 后端改进

#### 新增功能
- **文档处理能力**：支持 PDF 和 Word 文档解析
- **文档转换器**：`document_converter.py` 实现格式转换
- **批量处理优化**：改进的批处理配置

#### 依赖更新
```python
# 新增
"pypdfium2>=4.30.0",      # PDF 文档处理
"python-docx>=1.0.0",     # Word 文档处理
"python-multipart"        # 多部分表单数据
```

### 3. ⚙️ 配置改进

新增 `document_processing` 配置段：
```yaml
document_processing:
  enabled: true
  batch_size: 3              # VLM 批处理大小（推荐 2-5）
  max_image_size: 1024       # 最大图片尺寸（像素）
  dpi: 200                   # PDF 转图片 DPI（推荐 150-300）
  text_threshold_per_page: 50 # 扫描文档检测阈值
```

## ⚠️ 注意事项和已知问题

### 1. Python 版本要求 ⚠️

**变更**：Python 最低版本从 **3.9** 提升到 **3.10**

**影响**：
- 用户需要升级 Python 版本
- CI/CD 环境需要更新

**建议**：
```bash

```

## 📚 下一步行动

### 对于开发者

1. **安装依赖**
   ```bash
   # 前端
   cd frontend
   npm install

   # 后端
   cd ..
   uv sync
   ```

2. **运行开发环境**
   ```bash
   cd frontend
   npm run dev
   ```

3. **构建生产版本**
   ```bash
   cd frontend
   npm run build:win
   cd ..
   .\build-installer.bat
   ```

4. **测试安装程序**
   - 在干净的测试环境中安装
   - 验证所有功能
   - 检查开机启动

### 对于最终用户

1. **备份数据**（可选）
   - 备份 `~/Library/Application Support/OwlContext/Data`（macOS）
   - 备份 `%APPDATA%\OwlContext\Data`（Windows）

2. **卸载旧版本**（可选）
   - Windows：控制面板 → 程序和功能
   - 或使用卸载程序

3. **安装新版本**
   - 运行 `OwlContext-Setup-0.1.4-beta.exe`
   - 按提示完成安装

4. **验证安装**
   - 运行 `scripts\check-startup.bat` 检查开机启动
   - 启动应用并验证功能

5. **配置设置**（如果是新安装）
   - 输入 API Key
   - 配置录制源
   - 设置录制时间段
